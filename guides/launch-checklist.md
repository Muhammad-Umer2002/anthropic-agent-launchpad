# Launch Checklist (For Real Adoption)

## Before Publishing

- [ ] README explains value in first 10 seconds
- [ ] Repo includes copy/paste templates
- [ ] Safety and cost notes are present
- [ ] Clear "who this is for" section
- [ ] Source links are cited

## GitHub Setup

- [ ] Add clean repository description
- [ ] Add topics (ai, agents, anthropic, mcp, prompt-engineering, github-actions)
- [ ] Add MIT license
- [ ] Add social preview image
- [ ] Pin repository on profile

## Distribution

- [ ] Publish a LinkedIn post with problem + solution + repo link
- [ ] Share in relevant communities (Reddit, X, Discord, Dev.to)
- [ ] Add short demo GIF/video
- [ ] Ask first 10 users for feedback and iterate

## Growth Loop

- [ ] Convert common questions into FAQ
- [ ] Add one useful update per week
- [ ] Track stars, clones, issues, and forks
- [ ] Build in public with changelog posts
